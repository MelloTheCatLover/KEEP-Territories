# Style Guide — Образовательная платформа с геймификацией

Дизайн-система: **dark-first, спортивно-соревновательная, с glass-слоями**.
Бренд — **Byzantine Purple**. Референсы: Linear (дисциплина), Risk (логика карты), Tremor (данные).

---

## Философия

Платформа — это **турнирная админка**, а не игра. Интерфейс собран, холоден, читаем. Ощущение возникает за счёт точности, а не декораций:

- **Холодный графит** вместо чёрного фона — чтобы glass-слои читались и не давили
- **Один сильный акцент** (фиолетовый) работает как подсветка — никогда как заливка больших площадей
- **Плоские цвета команд** — карта наследует логику Risk: залил сектор, дорисовал бейдж сложности поверх
- **Тонкие светлые границы сверху** эмулируют отражённый свет — стандарт современных dark-интерфейсов (Linear, visionOS)

---

## Цвета

### Нейтрали

12 ступеней, холодный сине-графитовый уклон (hue ~220). Используй семантически:

- `neutral-0` — корень приложения, самый глубокий слой
- `neutral-50` — основной фон (body)
- `neutral-100` → `neutral-200` — поверхности (сайдбар, карточки)
- `neutral-300` → `neutral-400` — hover-фоны и границы
- `neutral-700` — вторичный текст
- `neutral-900` — основной текст
- `neutral-1000` — hero-заголовки, текст на акцентных заливках

**Правило:** никогда не используй `#000` или `#FFF`. Всегда через шкалу.

### Бренд — Byzantine Purple

- `brand-500` (#9D4EDD) — **основной рабочий**: CTA, активные состояния, иконки состояний
- `brand-400` — hover CTA
- `brand-600` — active/pressed CTA
- `brand-700+` — глубокие подложки, рамки выделения

**Правило:** бренд-фиолетовый никогда не лежит заливкой на карте. Он живёт только в UI-хроме. Это сохраняет отличие "UI-акцент vs цвет команды".

### Команды (8)

Классическая палитра, приглушённая до уровня насыщенности, удобного для плоской заливки. Порядок выдачи командам (максимальная разводка по hue):

`crimson → coral → amber → emerald → cyan → azure → indigo → magenta`

Каждая имеет три варианта:
- `base` — стандартная заливка сектора
- `bright` — выделенный/активный сектор (мой сектор, только что захваченный)
- `muted` — нейтрализованный (потерянный/неактивный сектор, фон в UI)

`textOnBase` указывает нейтраль для текста поверх заливки — используй её, не подбирай вручную.

**Проверка дальтонизма:** критичные пары (crimson/emerald, azure/indigo) разведены по светлоте и hue. При строгих требованиях AAA добавь паттерны (штриховка).

### Сложности

Температурная шкала: `easy` (мята) → `medium` (жёлтый) → `hard` (оранжевый) → `core` (малиновый).

**Правило:** сложность читается как **бейдж или иконка** поверх заливки команды, не конкурирует с цветом команды. Если бейдж попадает на близкий по hue цвет команды — используй `neutral-100` подложку под бейдж.

### Семантика

`success / warning / danger / info` — каждый с четырьмя вариантами: `bg` (фон алерта), основной, `hover`, `text` (текст на тёмном фоне).

**Не смешивай** семантику с брендом: ошибка — всегда красная, не фиолетовая.

### Контрасты WCAG AA

Все пары текст/фон основной палитры проверены:
- `neutral-900 / neutral-50` — 14.2:1 (AAA)
- `neutral-700 / neutral-50` — 5.8:1 (AA)
- `neutral-1000 / brand-600` — 8.1:1 (AAA)

## Типографика

**Rubik** (display) + **Inter** (UI) + **JetBrains Mono** (код).

Rubik заменил Space Grotesk из-за нативной поддержки кириллицы. Сохраняет геометричность и спортивный характер.

### Когда что

| Роль | Токен | Шрифт/вес |
|---|---|---|
| Hero, результаты матча | `display` / `heading-xl` | Rubik 700 |
| Заголовок страницы | `heading-md` | Rubik 600 |
| Заголовок карточки | `heading-sm` | Rubik 600 |
| Подзаголовок | `text-xl` | Inter 500 |
| Основной текст | `text-base` | Inter 400 |
| Вторичный текст | `text-sm`, `neutral-700` | Inter 400 |
| UI (кнопки, пункты меню) | `text-sm/base` | Inter 500 |
| Бейджи, лейблы | `.label` утилита | Inter 600 UPPERCASE, трекинг +0.05em |
| Цифры характеристик | `.numeric` утилита | Rubik 600, `tnum` |
| Код в задании | `code-block` | JetBrains Mono 400 |

### Правила

1. **Не смешивай** Rubik и Inter в пределах одной смысловой группы (заголовок карточки и цифра под ним — оба Rubik через `.numeric`).
2. **Моноширинный — только код.** Никаких "стилизованных" моно-подписей в UI.
3. **Tabular figures** обязательны для любых меняющихся чисел (очки, таймер, счётчики) — через `.numeric` или `font-variant-numeric: tabular-nums`.
4. **Никаких light-весов** (300). На тёмном фоне плохо читается.

## Отступы, радиусы

### Отступы — 4px база

- внутри компонента → `space-1` … `space-4`
- между компонентами → `space-4` … `space-6`
- между секциями → `space-6` … `space-8`

Если нужно "чуть больше" — **бери следующий шаг шкалы**, не изобретай 18px.

### Радиусы

- Инпуты/чипы — `xs` (4px)
- Кнопки — `sm` (6px)
- Карточки, панели — `md` (10px)
- Glass-контейнеры — `lg` (14px)
- Модалки — `xl` (20px)
- Аватары, pill-бейджи, icon-buttons — `full`

**Правило:** у вложенных контейнеров радиус уменьшается. Модалка `xl` → карточка внутри `md` → чип внутри `xs`.

## Glass

Glass — это **комплект из трёх параметров**, не просто прозрачность:

```css
.panel {
  background: var(--glass-bg-medium);
  backdrop-filter: var(--glass-blur);
  border: 1px solid var(--glass-border);
  box-shadow: var(--elevation-2);
  border-radius: var(--radius-lg);
}
```

Три уровня интенсивности:
- `subtle` — панели HUD поверх карты
- `medium` — карточки характеристик команд
- `strong` — модалки, dropdown

**Не переусердствуй.** Glass работает когда есть что показать сквозь — цветной фон карты, градиент. На чистом neutral-50 это просто менее тёмная панель. В списках и настройках — обычные `neutral-100/200` поверхности.

---

## Elevation

Стандартная логика: тёмная тень снизу + светлая внутренняя граница сверху (эффект отражённого света).

- `elevation-1` — статичная карточка
- `elevation-2` — hover
- `elevation-3` — dropdown/popover
- `elevation-4` — модалка
- `elevation-glow-brand` — **один активный элемент на экран** (мой сектор, приоритетная задача)

## Motion

- **Стандарт интерактива:** `duration-fast` (150ms) + `ease-out`
- **Появление панелей:** `duration-base` (220ms) + `ease-out`
- **Модалки:** `duration-slow` (320ms) + `ease-out`
- **Захват сектора на карте:** `duration-slower` (500ms) + `ease-in-out`

**Запрещено:** bounce, elastic, overshoot. Спортивный UI = собранность.

`prefers-reduced-motion` уже обработан в `tokens.css` — всё анимированное замирает при системной настройке.

---

## Чеклист для каждого нового компонента

1. Все цвета — через токены (`var(--color-*)`), не хардкод
2. Отступы — из шкалы, не произвольные
3. Радиус соответствует уровню вложенности
4. Есть hover, active, focus-visible, disabled состояния
5. Focus-ring использует `--state-focus-ring`
6. Цифры — через `.numeric` (tabular)
7. Лейблы — через `.label` (uppercase)
8. Проверен контраст на WCAG AA
9. Работает при `prefers-reduced-motion: reduce`

## Файлы системы

- `tokens.css` — CSS-переменные, импортируй в корне (`import './tokens.css'`)
- `tailwind.config.js` — если используешь Tailwind
- `design-tokens.ts` — константы для JS/TS (inline-стили, runtime-логика вроде выдачи цветов командам)
- `STYLE_GUIDE.md` — этот файл

При расхождениях источник истины — `design-tokens.ts`. Остальные файлы зеркалят его.
